Technical Design Document

Agentic SOP Search & Execution Platform

An extensible architecture built on Google Agent Development Kit (ADK) 2.x and OpenAI GPT-5.5. Provides capability-based semantic search and safe, deterministic execution of operational procedures.

Architectural Concept

Modern enterprises struggle with running Standard Operating Procedures (SOPs). Human prose documents are hard to search, and letting raw LLM agents execute them directly without guardrails poses significant operational risk.

Our platform introduces a Decoupled Architecture separating procedural knowledge from system integrations, and divides execution duties between a reasoning agent and a safe workflow runner:

📖

Knowledge Management

Process experts author procedures using human-readable steps mapped to high-level abstract capabilities (e.g. freeze_card), completely independent of code implementations.

  • Markdown metadata (SKILL.md)
  • YAML Execution Graphs
  • Strict Capability Lists
⚙️

Harness Orchestrator

Acts as the safety controller. It performs semantic searches, validates graph preconditions, dynamically binds abstract steps to code endpoints, and guarantees transaction-level rollback.

  • Dynamic Binding Registry
  • Precondition Validation
  • Human Gate approvals
🏦

Line of Business (LOB)

System engineers build concrete tools (microservices, DB operations) and map parameters. LOB interfaces can undergo refactoring and upgrades without impacting the KM team's SOPs.

  • Concrete Python Tools
  • Dynamic Param Translator
  • Version-insulated upgrades

Core Platform Concepts

To understand the system design, it is critical to distinguish between the two central pillars defined by the Google ADK: the Skill (design-time blueprint) and the Agent Harness (runtime execution safety controller).

📦

What is an ADK Skill?

In the Google Agent Development Kit (ADK), a Skill is the modular encapsulation of a specific domain capability. It represents the declarative contract and logical flow of a process.

  • Declarative Interface: Declared via a skill folder containing a SKILL.md. This file details the skill's name, description, and list of allowed-tools, acting as a strict security barrier for the Agent's runtime.
  • Executable Graph: Contains the workflow.yaml file, which specifies the state transitions, inputs, success checks, failure routes, and rollback actions.
  • Decoupled Focus: Authored by the Knowledge Management (KM) team. It contains no python imports, no API details, and no database schemas, remaining completely pure.
⚙️

What is the Agent Harness?

The Agent Harness is the stateful runtime orchestrator. Unlike raw, autonomous agents that plan dynamically and call APIs directly (introducing risks of hallucinations or infinite loops), the Harness acts as a deterministic safety container.

  • Precondition Guard: Verifies state criteria (e.g., customer account active) before enabling workflow progression.
  • Dynamic Binder Engine: Maps abstract YAML steps (like freeze_card) to concrete technical APIs (like lob_debit_card_freeze) at runtime, translating parameters on-the-fly.
  • Cognitive Delegation: For diagnostic steps, it instantiates an isolated ADK Agent (running on gpt-5.5) via the Runner and restricts its tools to the skill's allowed-tools.
  • Compensation & Rollback: Guarantees transactional integrity. If any step fails, the Harness executes rollback steps in reverse sequence (e.g. unfreezing a card on replacement dispatch failure).

Telemetry & Explainability Guardrails

To run autonomous processes in corporate environments, safety limits and clear audits are mandatory. Our platform implements two types of declarative guardrails inside the SOP execution pipeline:

📈

Telemetry Guardrails

The Harness continuously monitors system-level metrics and LLM usage to prevent runaway costs or high response latencies.

  • Global Budget Cap: Enforces a max_workflow_cost_usd (e.g. $0.10). The Harness tracks cumulative API usage fees and halts step execution immediately if the limit is breached.
  • Step Latency Gating: If a step exceeds its configured alert_on_latency_ms limit, the Harness generates alert warnings without failing the business flow.
  • Live Token Auditing: Captures ADK runner metadata (prompt, completion, and total tokens) to log accurate performance metrics.
🔍

Explainability Guardrails

Ensures that agent decisions are legally defensible and auditable. AI Agents cannot perform actions without generating structured reasoning logs.

  • Thought Stream Extraction: Taps into the ADK Agent's thoughts and reasoning path during diagnose steps.
  • Audit Justifications: Logs the raw decision-making context and populates it directly into the Harness context (e.g., fraud_justification).
  • HITL Context Injection: Automatically presents the generated explanation to human managers when prompting them for approvals at decision gates.

# Stateful execution engine incorporating Google ADK Agent
from models import AgenticSOP, SOPStep
from tools_registry import CAPABILITY_BINDINGS
from google.adk import Agent, Runner
from google.adk.sessions.in_memory_session_service import InMemorySessionService

class HarnessAgent:
    def __init__(self, workflow_path: str):
        self.workflow_path = workflow_path
        self.sop: Optional[AgenticSOP] = None
        self.context: Dict[str, Any] = {}
        self.executed_steps: List[str] = []

    async def execute_step(self, step: SOPStep) -> bool:
        start_step_time = time.time()
        
        # 1. Resolve template inputs and dispatch
        resolved_inputs = {k: self._resolve_template(v) for k, v in (step.inputs or {}).items()}
        if step.action == "execute":
            success = await self._run_execute_action(step, resolved_inputs)
        elif step.action == "diagnose":
            success = await self._run_diagnose_action(step)
        elif step.action == "decision":
            success = await self._run_decision_action(step, resolved_inputs)
        
        # 2. Latency Telemetry Gating
        duration_ms = int((time.time() - start_step_time) * 1000)
        if step.telemetry and step.telemetry.alert_on_latency_ms:
            if duration_ms > step.telemetry.alert_on_latency_ms:
                logger.warning(f"⚠️  [TELEMETRY ALERT] Step '{step.id}' latency {duration_ms}ms exceeded limit!")
        return success

    async def _run_diagnose_action(self, step: SOPStep) -> bool:
        # Spawn cognitive ADK Agent using gpt-5.5
        agent = Agent(
            name="FraudDetectorAgent",
            model=os.environ.get("OPENAI_MODEL", "gpt-5.5"),
            instruction=f"Bank fraud analysis. {step.instruction}",
            tools=[get_transaction_history]
        )
        runner = Runner(agent=agent, session_service=InMemorySessionService(), auto_create_session=True)
        
        # Stream and capture token counts
        async for event in runner.run_async(...):
            if event.usage_metadata:
                prompt_tokens = event.usage_metadata.prompt_token_count
                comp_tokens = event.usage_metadata.candidates_token_count
                # Estimate cost
                step_cost = (prompt_tokens * 0.000015) + (comp_tokens * 0.000060)
                self.context["accumulated_cost_usd"] = self.context.get("accumulated_cost_usd", 0.0) + step_cost
        
        # Explainability Tap: Capture thought path
        if step.explainability and step.explainability.required:
            self.context[step.explainability.target_field] = agent_response_text
        return True

    async def run(self, initial_inputs: Dict[str, Any]) -> bool:
        self.context = dict(initial_inputs)
        self.load_workflow()
        
        for step in self.sop.steps:
            # Enforce budget guardrails
            current_cost = self.context.get("accumulated_cost_usd", 0.0)
            if self.sop.global_governance and current_cost > self.sop.global_governance.max_workflow_cost_usd:
                logger.error(f"❌ [GOVERNANCE BREACH] Cost ${current_cost} exceeds limit!")
                return False
                
            success = await self.execute_step(step)
            if not success:
                if step.on_failure == "goto step_rollback":
                    await self.execute_rollback()
                return False
        return True

class SOPStore:
    def __init__(self, base_dir: str = "."):
        self.base_dir = os.path.abspath(base_dir)

    def search(self, query: str, agent_capabilities: List[str]) -> List[Dict[str, Any]]:
        """
        Capability-Aware Search:
        1. Filters out SOPs whose allowed_tools are not satisfied by agent_capabilities
        2. Scores name and description relevance using keyword overlap
        """
        all_skills = self.scan_skills()
        matching_skills = []
        query_words = set(re.findall(r"\w+", query.lower()))

        for skill in all_skills:
            # Filter by capability overlap
            required_tools = skill.get("allowed_tools", [])
            if not all(tool in agent_capabilities for tool in required_tools):
                continue

            # Compute match score
            score = sum(5 if w in skill["name"].lower() else (2 if w in skill["description"].lower() else 0) for w in query_words)
            if score > 0 or not query_words:
                skill_copy = dict(skill)
                skill_copy["search_score"] = score
                matching_skills.append(skill_copy)

        matching_skills.sort(key=lambda x: x["search_score"], reverse=True)
        return matching_skills

import openai.resources.chat.completions
original_create = openai.resources.chat.completions.AsyncCompletions.create

# Monkey patch OpenAI client completions for GPT-5.5 compatibility
async def patched_create(self, *args, **kwargs):
    if "max_tokens" in kwargs:
        model = kwargs.get("model", "")
        if "gpt-5.5" in model or model.startswith("o1") or model.startswith("o3"):
            kwargs["max_completion_tokens"] = kwargs.pop("max_tokens")
    return await original_create(self, *args, **kwargs)

openai.resources.chat.completions.AsyncCompletions.create = patched_create
